Meeting to Merge preserves the path from a spoken requirement to a proposed code change. The fixture contains four timestamped requirements and an explicit definition of approval. Running the analysis reproduces four baseline failures, verifies four patched behaviors, then produces a minimal unified diff covering input validation, approval, receipt identity, and idempotency. The patch stays behind a human apply boundary. I used Codex with GPT-5.6 to translate intent into checks, implement and regression-test the CRLF fix, and review the human apply boundary. A live GPT-5.6 extension can draft requirements from longer transcripts, while executable checks remain authoritative.
